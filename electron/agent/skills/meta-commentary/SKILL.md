此 skill 定义和阐述对白优化中最主要的技巧：**元评论**（Meta-Commentary）。

## 核心原则

**对白不止传递信息，还要让角色对"当前处境"和"对方刚说的话"做出反应和评价。人物不应该只是"说话"——他们应该一直在"评论着自己正在说的话和正在发生的事"。**

大多数新手对白的问题不是"信息不对"——而是"只有信息"。角色像新闻播报员一样交换需要传递的情节数据，然后对话结束。元评论让对白从"信息播报"升级为"人物展示"。每一句纯信息交换之后，至少应该有一句带有元评论色彩的反应。

**铁律：元评论必须说出来。** 元评论是对白的一部分——是角色对在场（或不在场）的另一个人说出的话。它不是内心独白，不是无声的心理活动，不是观众才能听到的吐槽。它必须从角色的嘴里发出声音，被场景中的其他人听到——哪怕那个人没有回应。如果元评论只存在于角色的脑子里而没有变成声波——那就不是元评论，那是心理描写。

**铁律二：元评论必须有对话对象。** 嘟囔、自言自语、对着空气说话——这些是元评论的末流形态，占比必须尽量小，即使有也要极其简短（不超过半句话的长度）。有效的元评论是**嵌入在对白中的**——有明确的对话双方，能引起对方的注意或情绪反应（对方可能反驳、沉默、皱眉、笑——只要有反应即为有效）。唯一例外：**角色对着镜子里的自己说话**——镜像是"另一个自己"，构成了对话双方，属于有效添加。

## 元评论的五种形态

### 1. 自嘲
角色对自己处境的幽默化评价。注意：不是"作者替角色吐槽"，而是角色本人的幽默感在面对自己的处境。

**自嘲必须对另一个人说。** 不要让对方沉默——让自嘲成为抛给对方的一个球。对方可能接（反驳、笑、翻白眼）也可能不接（沉默）——但沉默本身必须是一个可见的反应。嘟囔、自言自语式的自嘲占比必须尽量小（不超过元评论总数的10%），即使有也要极其简短（半句话以内）。唯一例外：角色对着镜子里的自己说话——镜像构成对话双方，自嘲在这里是"我对我自己的审视"。

×（作者吐槽）："我真是太倒霉了。"
✓（角色自嘲）："我就知道会这样——我的倒霉应急包里没有'应急包丢了怎么办'这个选项。"

### 2. 点评对方言论
在回答问题之前，先评价对方"问了什么"或"怎么问的"。不是回避问题，而是在回答问题之前先给对方的提问方式打一个标签。

弱（直接回答）：警官："你昨天晚上八点在什么地方？" 嫌疑人："在家。"
强（先点评再回答）：警官："你昨天晚上八点在什么地方？" 嫌疑人："（轻笑）你们每次开场都这句，不腻吗？……在家。"

### 3. 定义当下处境
角色对"正在发生的事"给出自己命名的标签。这个标签是角色对事件的个人版本解读。

"所以这就是我的告别派对——在一个面包店里，以社死的形式。"

### 4. 夸赞中的暗刺
表面上在夸，但听者——以及观众——都能感受到被夸的东西其实是一把刀。

"师傅，小文是真心希望你好的——真的。"

### 5. 沉默作为一种元评论
角色选择不说话——但观众知道这个沉默意味着什么。在没有被填满的空白中，角色的"不回答"本身就是最响亮的回答。沉默在以下时刻效力最强：当观众确切知道角色在沉默中隐瞒的是什么、承受的是什么、或拒绝的是什么。

**注意：沉默是"选择不开口"，不是"在心里想"。** 沉默是外在的、可见的、能被摄影机捕捉的——观众看到角色选择了闭嘴。它与内心独白完全不同——内心独白是角色在心里说的话，观众通过画外音听到；沉默是角色在外部世界中的行为。如果角色在心里说了元评论但没有发出声音——那不是元评论，那是内心独白。

## 频率原则

元评论的使用频率是所有对白技巧中最高的。大多数对白线在从 Phase 1（功能正确）进入 Phase 2（技巧注入）时，第一步就应该是：**给这句纯信息的台词加一层元评论的外壳。**

不是每句都需要。但当连续出现三句以上"纯信息交换没有元评论"的对白时，这段对话开始平面化。

## 元评论 ≠ 插科打诨

元评论不是"让角色变幽默"。严肃角色的元评论可以很重——

弱（纯信息传递）："我知道他会死的。"
强（加入元评论，严肃角色的评价）："我知道他会死的。我只是从来没能把这个'知道'变成——你永远没办法把一个事实变成能咽下去的东西。"

元评论的本质是"角色的态度"如何在语言中渗出，而不是"角色的笑话"。

## 与同组技巧的关系

- 与 [[qa-role-reversal]] 配合 —— 元评论常作为角色翻转问答模式的前奏（先评价你的问题，然后回答我自己的问题）
- 与 [[subtext-first]] 配合 —— 元评论是潜台词的载体之一：角色用评论来包装自己真正想说但不敢直说的话
- 与 [[callback-echo]] 配合 —— 回声后的元评论可以让一句被回收的台词获得全新的重量
