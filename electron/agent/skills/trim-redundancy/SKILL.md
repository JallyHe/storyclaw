此 skill 定义和阐述对白优化原则之一：**删减冗余**（Trim Redundancy）。

## 核心原则

**写完第一稿后，尝试删掉每句对白的前半句和后半句，看核心意思是否还在。所有开场白、客套话、重复内容、和可以用动作表达的对白——一律删除。**

大多数对白中的冗余不是"垃圾信息"——是创作者在写第一稿时用来"热身的"语言。它们帮助创作者进入了对话的节奏，但对观众来说——这些字就是纯粹的噪音。删减是"把对话从铅笔稿变成墨线"的过程。

**铁律：删减删除的是冗余表达，不是信息本身。** 删掉"你好"和"我叫你一声老王"——不影响信息。但如果删掉了一句话后，观众会缺少某个情节条件、动机线索或铺垫信息来理解后续剧情——那就不是冗余，那是对白在负载信息。**删减前先问：这句话在传递什么信息？这个信息在其他地方是否已经存在？删掉后观众是否仍然拥有理解后续剧情所需的全部信息？**

## 删减测试

对每一句台词执行以下测试：

**前删测试**：删掉这句话的前半句。核心意思还完整吗？
**后删测试**：删掉这句话的后半句。核心意思还在吗？
**全动作测试**：这句话能用一个动作或一个眼神替代吗？
**信息完整性测试**：这句话负载了任何情节推进、角色动机、关系变化、或伏笔铺垫信息吗？如果有，这个信息在其他地方是否已经被传递过了？如果答案分别是"是"和"否"——这句话不能删。

如果前三个测试中任何一个得到"可以删"——且信息完整性测试通过——就删。

## 四类必删冗余

### 1. 开场白和客套话
- ×（删前）："你好。""最近怎么样？""我跟你说个事儿……"
- 删这些的话不是"对话不礼貌了"——是"对话更精准了"。生活中需要客套，剧本不需要。

### 2. 角色互相叫名字
- ×（删前）：在每句话前面加对方的名字。"老王，我跟你说……""不对，小刘，你听我说……"
- 影视对白不是现实对话。观众从上下文知道谁在跟谁说话。名字只在称呼不明确或名字本身负载潜台词时才保留。

### 3. 重复前一句话的内容
- ×（删前）：A："我昨天去了医院。" B："你昨天去了医院？"
- 除非B的重复负载了潜台词（震惊、质疑、讽刺），否则删掉。

### 4. 角色"解释"已经发生的事
- ×（删前）：角色A对角色B详述了一件观众已经看到的事情。
- 观众不需要听两遍。如果角色B需要知道这个信息——要么用动作展示B看到/得知，要么A只用一个短句告知结果，省略所有观众已知道的细节。

## 动作替代对白

**如果一个行动可以传递的信息，不要用台词去说。**

| 删除（台词） | 替代（动作） |
|-------------|------------|
| "我走了。" | △ 拿起钥匙。门关上了。 |
| "我很难过。" | △ 她的手搭在窗台上，手里那根烟没点——就一直那么拿着。 |
| "我不相信你。" | △ 他把合同推到一边。没有碰。 |
| "我等了很久了。" | △ 桌上三个空咖啡杯。 |

## 删减后的回填检查

删完一遍之后，重新通读。两个维度都要检查：

**信息丢失检查（优先）**：逐一核对——原作中的情节条件、角色动机、关系节点、铺垫信息是否都还在？如果删掉某句台词后，后续某场戏的理解链条出现了断点——该句台词在负载信息，必须回填（最小体量）。

**节奏断裂检查**：有些你删掉的东西可能真的在负载不易察觉的节奏功能——删完之后对话的节奏断了、缺少了一个必要的节拍。这时把它们——最小体量地——回填进去。

回填的原则是——只加回"删了之后对话确实坏了"的那最小量。信息丢失的优先级高于节奏断裂——丢了信息是事故，断了节奏是可以后续用 [[rhythm-control]] 修复的问题。

## 与同组技巧的关系

- 与 [[subtext-first]] 配合 —— 删掉直说的，剩下的才有潜台词的空间
- 与 [[action-catalyzing]] 配合 —— 删掉可以用动作替代的对白后，剩下的对白和动作之间的张力更强
- 与 [[rhythm-control]] 配合 —— 冗余是节奏的死敌。删减后的对话自然获得了更紧凑的节奏
