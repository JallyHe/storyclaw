此 skill 定义和阐述"D 情节"（Situation-Building Plot，构建情境的情节）——故事结构体系中负责建立和强化故事情境的情节单元。对应原有 story-restructurer skill 中的"构建情境的情节"（Layer C）。

## 核心定义

**D 情节是通过一段情节的发展，改变故事情境（C）的压力状态的情节单元。它有两种基本形态：前置 D 情节（在 C 建立之前，将故事推入 C）和内置 D 情节（在 C 运行之中，使 C 的压力骤然增强）。**

## D 情节的两种形态

### 形态一：前置 D 情节（Pre-C D-Plot）—— 情境的入口

前置 D 情节将故事从"尚未进入情境"的状态推入一个明确的 C。在它发生之前，故事可能还处于日常状态、前一个情境的收束阶段、或情境尚未被清晰定义的模糊区域。前置 D 情节的执行让故事获得了一个明确的"重力中心"——观众和角色同时意识到："从这一刻起，事情不一样了。"

前置 D 情节回答三个问题：
1. 为什么这个情境对主角重要？
2. 主角为了进入这个情境，付出了/牺牲了什么？
3. 如果情境不被解决，具体的代价是什么？

### 形态二：内置 D 情节（Intra-C D-Plot）—— 情境的增压器

内置 D 情节出现在 C 运行的过程中，穿插在 A 情节之间或 A 情节内部。它不开启新的 C——它让当前的 C 变得更紧张、更危险、压力更大。内置 D 情节的发生让主角在原有的困境之上，被施加了一个额外的、往往是意料之外的负担。

**比喻**：C 是主角必须走过两栋高楼之间的一根钢丝绳。A 情节是主角在钢丝上完成的动作——往前走、翻跟头、转圈——这些是她为了到达对面而做的尝试。内置 D 情节则是突然刮来的一阵狂风，或是对面楼上出现了一个拿着锯子正要锯断钢丝的人。钢丝绳本身没有变——但走钢丝的难度骤然升级了。

内置 D 情节的关键特征：
- 它不改变 C 的定义——C 仍然是同一个 C
- 它不解决 C——它让 C 变得更难被解决
- 它的出现是意料之外的——主角（和观众）没有预料到这个增压事件会发生
- 它的作用是让后续的 A 情节在更高的压力条件下进行

内置 D 情节回答的问题：
1. 在这个 C 里，还有什么能让主角的处境变得更糟？
2. 这个新增的压力来源是否与 C 的核心张力一致？（如果不一致——它是噪音，不是 D 情节）

### 两种形态的关系

前置 D 情节是"推入"——把故事推进一个 C。内置 D 情节是"拧紧"——在 C 内部的某个节点上，把已经存在的压力再拧紧一圈。一个完整的 C 可以有一个前置 D 情节和零到多个内置 D 情节。并非每个 C 都需要内置 D 情节——只有当 A 情节的压力递增需要被一个外部事件加速时，才需要调用它。

## D 情节的结构

### 阶段一：触发事件（Trigger Event）
一个事件打破了角色的日常平衡。这不一定是一个灾难——它可以是一封信、一个电话、一个意外的发现。关键是：这个事件让角色意识到"有某种未解决的状态"。

### 阶段二：投入（Commitment）
角色做了一个选择——进入这个情境。不是"被卷入"，而是选择了介入。即使这个选择是被迫的，角色也必须表现出"这是我要做的事"的主动姿态。

### 阶段三：确认赌注（Stake Confirmation）
让观众明确感知到"如果失败了会怎样"。这可以通过：
- 展示一个此情境不被解决时的"预演"画面
- 让一个第三方角色说出"你不解决这个问题的代价是……"
- 角色自己做出了一个"为了这个情境我愿意放弃 X"的宣言或行动

### 阶段四：情境确立（Situation Lock）
情境正式被锁定。一个标志性的画面或台词，让观众在后续的 A 情节中可以回忆：故事是从这里正式开始的。

## D 情节的常见模式

### 损失驱动型
角色失去了某样重要的东西。进入情境是为了"挽回"或"找到真相"。

### 威胁降临型
一个外部威胁即将到来。进入情境是为了"准备"或"阻止"。

### 选择后果型
角色之前做出的一个选择产生了延迟后果。进入情境是为了"承担"或"修正"。

### 发现驱动型
角色发现了之前不知道的信息。进入情境是因为"知道了这件事就不能装作不知道"。

## D 情节与角色投入的深度

D 情节的质量取决于"角色为什么在乎"的说服力：

- **浅层投入**：角色在乎是因为"应该在乎"（职业要求、道德义务）
- **深层投入**：角色在乎是因为"这个情境触动了一个个人化的旧伤或未竟之事"

好的 D 情节让两者并存——表层理由让角色"开始行动"，深层理由让角色"无法停止"。

## D 情节的判断标准

1. 观众是否在 D 情节结束时明确知道"这个故事的引力中心是什么"？
2. 角色进入情境的选择是否具有不可逆性？—— 如果角色随时可以退出，情境没有真正建立。
3. 代价（赌注）是否被具体化了？—— 不能只是"会失败"，要说清楚"失败意味着什么"。

## 常见问题

### D 情节过长
如果构建情境花了太长的时间，观众会在情境建立之前就失去耐心。D 情节应该被压缩到"刚好够让观众理解"。

### D 情节与 A 情节混淆
D 情节不包含"尝试解决情境"的步骤——那是 A 情节的工作。前置 D 情节只负责"进入情境"，内置 D 情节只负责"增压情境"。两者的共同点是：都不尝试解决任何东西。

### 内置 D 情节与 A 情节的复杂化混淆
A 情节的"复杂化"是主角自身行动的后果——"我做了什么导致了什么"。内置 D 情节的"增压"是来自外部的、不被主角控制的——"发生了什么让我的处境更难了"。区分标准：如果这个增压事件是主角在 A 情节中的行动引发的连锁反应——这是复杂化，属于 A 情节。如果这个增压事件是独立于主角行动的外部事件——这是内置 D 情节。

### 内置 D 情节与 B 情节混淆
内置 D 情节让 C 更难被解决，但它不终结 C。B 情节终结 C。一个 C 内部可以有一个内置 D 情节，但只会在末尾有一个 B 情节。

### 代价模糊
"如果失败就是世界末日"——这不是有效的代价。代价必须是角色个人的、具体的、观众能感受到疼的。

## 与故事结构体系其他概念的关系

- **角色情境** ← D 情节进入新情境时，角色的底色不会被"重建"——它一直在，只是被新情境重新照亮
- **故事情境（C）** ← 前置 D 情节是情境的入口；内置 D 情节在情境运行中拧紧压力的螺丝
- **A 情节** ← 前置 D 情节结束后，A 情节开始在情境内部运作；内置 D 情节穿插在 A 情节之间或内部，让后续 A 情节在更高的压力条件下进行
- **B 情节** ← D 情节（无论前置或内置）不终结情境——那是 B 情节的工作
- **故事线** ← 一条故事线上的每个关键情境都有各自的 D 情节序列（前置 + 零到多个内置）

## 使用说明

当用户在创作中需要：
- 设计一个故事的"入口"——如何把观众从日常状态带入戏剧状态
- 确保观众理解"为什么主角不能转身离开"
- 检查开篇的铺垫是否足够但不过度
- 识别故事进入情境的时间点是否合适
