此 skill 定义和阐述"B 情节"（Terminal Plot，终结情节）——故事结构体系中决定情境切换的过渡情节单元。对应原有 story-restructurer skill 中的"终结情节"（Layer E）。

## 核心定义

**B 情节是导致当前故事情境结束、并开启下一个故事情境的情节。**

在 B 情节中，维持当前情境的那个"不平衡"被一个事件或行动打破。旧的张力不再持续——不是因为被解决了，就是因为被转换为一种新的形态。而就在旧情境消散的同时，一个新的情境的种子被埋下。

B 情节是两个情境之间的桥梁和切换器。它是故事中最关键的结构节点——在此之前，A 情节反复尝试但未能改变情境；在此之后，所有角色面对的问题已经不再是同一个问题了。

## B 情节的结构要求

### 1. 尽可能短
B 情节应该"尽可能短"——一个行动、一次揭示、一句对话。它不是一场需要铺陈的大戏，而是一个精确的引爆点。引爆之后，气压已经变了——不需要再解释气压为什么变。

### 2. 旧情境必须被"改变"而非"遗忘"
B 情节不能只是"跳到一个新话题"——它必须让观众明确感知到：之前的那个状态不存在了。可以通过：
- 冲突的某一方被移除（死亡、离开、放弃）
- 某条关键信息的揭示改变了所有角色的认知
- 一个不可逆的行动让"回到之前"变得不可能

### 3. 新情境的种子必须被埋下
旧情境结束的那一刻，新情境必须被暗示或直接开启。新情境可以是一句话、一个画面、一个角色做出的新决定——但必须让观众产生新的追问。新情境不需要展开——它是种子，不是树。

## B 情节与新情境的关系

| B 情节的结束方式 | 新情境的类型 |
|------------------|-------------|
| 问题被解决 | 新的问题浮现（信息型→冲突型） |
| 对手被击败 | 更大的对手显露（冲突型→冲突型升级） |
| 秘密被揭示 | 揭示引发了新的不平衡（信息型→目标型） |
| 角色做出不可逆的选择 | 选择的后果成为新的外部压力（目标型→存在型） |

关键原则：B 情节不能只是"关掉一扇门"——必须同时"打开一扇窗"。观众在旧情境消散时的第一反应是"哦……"，紧接着必须是"那接下来呢？"

## B 情节的常见形态

### 揭示型 B 情节
一条被悬念保留的信息在 B 情节中被释放——揭示本身改变了所有角色的位置。角色的认知被重置，旧情境的"不知道"变成"知道"，张力瞬间转型。

### 行动型 B 情节
一个角色做出了一个不可逆的行动——这个行动改变了情境的条件，让原有的问题不再有意义。常见于动作/战斗题材：最终一战。

### 对话型 B 情节
一场对话中，某个真相被说出，或某个立场被确凿地摊牌——对话本身是行动，它改变了角色之间的关系结构。

### 外部事件型 B 情节
一个不受角色控制的外部事件发生——它打破了现有情境的所有前提条件。但需注意：外部事件必须与之前的 A 情节有因果联系，不能是"天降神兵"。

## B 情节的判断标准

1. 旧情境是否真的结束了？—— 观众不会再问之前那个问题了。
2. 新情境是否已经启动？—— 观众开始问一个新的问题了。
3. B 情节本身的长度是否被控制到了最小？—— 越长越稀释冲击力。
4. B 情节是否与前面的 A 情节有因果连接？—— 不是"然后"，是"因此"。

## 与故事结构体系其他概念的关系

- **角色情境** ← B 情节切换情境，但角色的永久底色不随之切换——旧情境结束了，角色还是那个人
- **故事情境（C）** ← B 情节结束旧情境，开启新情境
- **A 情节** ← 在 B 情节之前发生，它们铺垫了 B 情节的必要性
- **D 情节** ← B 情节开启新情境后，新情境的构建可能需要 D 情节来解释
- **故事线** ← 多个 B 情节串联，标记一条故事线的关键转折点

## 使用说明

当用户在创作中需要：
- 设计一个故事状态到另一个状态的转折
- 确保结局不是"突然结束"而是"有根的转换"
- 平衡 B 情节的"短促"与"冲击力"
- 在剧集的结尾埋下下一集的种子（cliffhanger 或 hook）

调用此 skill。
