# 故事架构五层详解（Architecture Detail）

本文件是 `story-restructurer` 技能的配套深读材料，完整说明 SKILL.md 中的五个架构层（A–E）及其相互关系。这是 StoryClaw ABCD 体系在「重构」语境下的展开。

> 一句话总览：**A、B 是两种「情境」（状态），C、D、E 是三种「情节」（动作）。情节的存在意义，永远是服务于情境的构建、循环或终结。**

---

## Layer A · 故事情境（Story Situation）
**定义**：定义「这一个」故事的特定张力，**会被解决**。绑定具体事件，有开始有结束。观众追问的是「**这会被解决吗？**」

- 是整个故事的引力中心（呼应原则 2）。
- 必须可被一句话说清，且含明确赌注。
- 一部长作品可串联多个故事情境（一个解决会种下下一个，见 Layer E）。

**示例**：「林姐能否在身份暴露前，确认母亲临终是否真的找过她？」——有具体事件（确认遗言真假）、有结束条件（确认/被揭穿）、有赌注（一旦暴露，全家翻脸、真相永埋）。

**重构要点**：先把故事情境钉死，其余一切围绕它排布。若一段内容不指向它，砍或并。

---

## Layer B · 角色情境（Character Situation）
**定义**：角色的**永久**戏剧处境，**不会被解决**。绑定角色本身，超出本故事而持续存在。观众追问的是「**这次它会怎样表现出来？**」

- 是人物的不变内核（性格 × 与世界的固定冲突模式）。
- 它不随剧情解决，只在每个故事情境里以新方式重复显现（呼应 character-reinforcement）。
- A 与 B 的张力交叉，产生人物在本故事中的具体选择。

**示例**：林姐的角色情境是「她永远在‘确认自己是否被爱’与‘害怕答案是否定’之间挣扎」。这不会因为本剧结束而消失；它在「确认母亲遗言」这个故事情境里，表现为「明明能一查到底，却屡屡在关键处退缩」。

**重构要点**：为每个主角写一句话角色情境，并检查 A 故事情境是否恰好「逼」这个 B 内核现形——若两者无关，故事会松。

---

## Layer C · 构建情境的情节（Situation-Building Plot）
**定义**：确立**为什么故事情境重要**。回答：这件事为何对主角至关重要？她为走到这一步牺牲了什么？失败的具体代价是什么？

- 体量要**克制**——构建是为了让后面的循环有分量，不是自我膨胀。
- 把抽象赌注变具体（用一个可见的牺牲/代价来锚定）。

**示例**：用两三个节拍交代——林姐为查真相，辞掉城里工作、隐姓埋名、烧掉自己的身份证明应聘保姆。观众由此明白：她已**无退路**，故事情境的赌注是真实的。

**重构要点**：构建情节常是重构中最该被压缩的部分。把「介绍背景」改写为「展示代价」，能短则短。

---

## Layer D · 循环情节（Cyclical Plot）
**定义**：为解决故事情境而**反复尝试**、并在过程中**升级压力**。每个循环：**尝试 → 部分成功 → 复杂化 → 比之前更糟的处境。**

- 是故事的主体量。一个故事情境内通常 2–4 个循环（张力强的可多）。
- **复杂化的来源必须是角色的性格缺陷或信息缺失，不能是「运气不好」。**
- 相邻循环不能同套路——每次「更糟」的方式要有区分度。
- 全程守原则 3：压力只增不减。

**单个循环示例（林姐第一次尝试查日记）**：
- 尝试：她趁全家外出潜入书房找母亲遗物。
- 部分成功：找到一本日记。
- 复杂化：关键几页被撕走（信息缺失）；且她的潜入被监控拍下（埋下新隐患）。
- 更糟：她不但没得到答案，还多了「可能暴露」的新压力。→ 进入下一个循环，压力更高。

**重构要点**：把原故事里散乱的事件，重排成「一串处境单调变糟的循环」。检查每个循环的复杂化是否源于人物/信息（合格）还是巧合（不合格，需改写）。

---

## Layer E · 终结情节（Terminal Plot）
**定义**：**解决**当前故事情境，并**确立新的**故事情境。应**尽可能短**——一个动作、一句话。新的故事情境是**一颗种子，不是一棵树**。

- 解决要有代价，不能轻松翻篇（呼应 pressure-escalation 的收束）。
- 种下的新故事情境只需一句话（呼应原则 6「一句话的坑」）。
- 在长剧里，E 同时是上一段的句号与下一段的逗号。

**示例**：林姐终于在老爷子病床前对质，确认「母亲当年确实找过她、是被拦下的」（解决故事情境）。同一场结尾，老爷子塞给她一把钥匙、只说「还有一个人，你该去见」——一句话种下新的故事情境（那个人是谁？），全剧转入下一段。

**重构要点**：找出原故事里冗长的「大结局」，把它压成「一个决定性动作 + 一句话新钩子」。若终结情节超过一两个节拍，多半在拖。

## 五层如何协作（重构装配顺序）
```
B 角色情境（不变内核）
        ↕   A×B 交叉 → 产生人物的具体选择
A 故事情境（本故事要解决的张力）  ← 引力中心
        │
   C 构建情境  →  D 循环×N（压力单调递增）  →  E 终结（解决+种下新A）
   (为何重要)     (反复尝试、越来越糟)          (一动作/一句话)
```

**装配步骤**：
1. 钉死 A（故事情境），写清赌注与结束条件。
2. 为每个主角写 B（角色情境），确认 A 恰好逼 B 现形。
3. 用尽量少的节拍写 C（构建情境＝展示代价）。
4. 把原事件重排成 2–4 个 D 循环，逐个核对「尝试→部分成功→复杂化（源于人物/信息）→更糟」，并保证压力单调递增。
5. 写一个极短的 E：一个动作解决 A，一句话种下新 A。
6. 通读：每一层是否都在服务 A？不服务的，删或并（原则 2）。

## 常见架构病与诊断
| 症状 | 病因 | 处方 |
|------|------|------|
| 中段拖沓、看不下去 | D 循环复杂化靠巧合，或相邻循环同套路 | 把复杂化改挂到人物缺陷/信息缺失；给每个循环不同的「变糟方式」 |
| 高潮没劲 | C 构建太弱，赌注没立住 | 在 C 加一个可见的牺牲/代价 |
| 谜底泄得太早、后程乏力 | E 提前、或某个 D 里解决了 A | 推迟解决，把「接近真相」改成「陷入更大压力」（原则 3）|
| 结局拖沓 | E 写成了「一棵树」 | 压成一个动作 + 一句话新钩子 |
| 人物像工具人 | A 与 B 无关，选择不从内核出 | 重定 B，使 A 恰好逼其现形（角色每个动作都应来自永久困境或两难）|
