# 剧本格式化

**定义**：把任意来源（粘贴、导入的 PDF/DOCX/TXT、其它软件导出）的剧本文本，归类为「场次头 / 对白 / 描述（动作）/ 转场」四类，转换成 StoryClaw 的 .ep 轻标记格式。系统内置一个**格式模板库**（每类预设若干正则规则），并能在遇到没见过的格式时由你新造规则、保存复用。

## 触发时机
- 用户要求「格式化/整理这段剧本」「把这个剧本转成标准格式」
- 用户粘贴了一段排版不规范的剧本
- 导入剧本后格式没对齐，需要重新整理

## 工作流（务必按此顺序）

1. **取文本**：拿到要格式化的原文（用户粘贴的，或用 read_reference/read_screenplay 读取的）。文本很长时，可先取前 60~100 行用于格式判断。

2. **匹配模板**：调用 `match_screenplay_format(text)`。
   - 它会给每个模板打分并给出建议。

3. **命中可信**（工具回复 ✅）：直接调用
   `apply_format_template(text=全文, title=标题, templateId="推荐的id")`，
   查看返回的统计与「仍被判为描述的样例」。没问题就用 `write_screenplay` 写入对应 .ep 文件（Craft 模式）。

4. **未命中**（工具回复 ⚠️）：你来**新造规则**。
   - 观察这段文本：场次头怎么写（如「第3场」「3.」「INT. 咖啡馆」「场景3」）、对白怎么写（如「张三：…」「张三（笑）：…」「张三 说：“…”」）、转场怎么写。
   - 为 scene / transition / dialogue 各写 1~N 条正则。**dialogue 正则必须含 2 个捕获组：组1=人物（可含括号说明），组2=台词**。
   - 调用 `apply_format_template(text, title, template={ name, scene:[...], transition:[...], dialogue:[...] })` **试跑**。
   - 看返回统计与「仍被判为描述的样例」：若样例里还有明显的对白/场次/转场被漏判，**修正正则再试**，直到结构化占比合理、漏判很少。
   - 满意后调用 `save_format_template(name, scene, transition, dialogue, description)` 保存。
   - 最后用保存返回的 id 再 `apply_format_template(...)` 得到最终结果，`write_screenplay` 写入。

5. **下次复用**：保存过的模板会进入模板库，之后遇到同类格式 `match_screenplay_format` 会直接命中。

## 核心约束
- 只做**格式归类与转换**，不改写剧情内容、不增删台词。
- 场次头输出 `# …`，转场输出 `> …`，对白输出 `人物：台词`（人物可带括号说明），其余作为描述段落原样保留。
- 写入 .ep 时必须保留/补全文件头 `@episode`、`@title`、`@status`。
- 自造的对白正则一定要有 2 个捕获组，否则对白会被当作描述。
- 正则要尽量精准，避免把描述段落误判成对白（例如不要用过于宽松的「任意冒号」规则）。

## 自检清单
- [ ] 先 match 再决定套用或新造

- [ ] 新造模板已试跑并验证，dialogue 含 2 个捕获组
- [ ] 新格式已 save_format_template 保存
